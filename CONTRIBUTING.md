# Contributing

Contributions are welcome! Here's how to get started.

## Development setup

### Prerequisites

- Node.js 18 or later
- npm
- Git (with submodule support for custom SDK builds)

### Getting started

```shell
# Clone the repository
git clone https://github.com/dxdc/aws-sdk-google-apps.git
cd aws-sdk-google-apps

# Install dependencies
npm install

# Run the build
npm run build

# Run tests
npm test
```

## Available scripts

| Script                  | Description                              |
| ----------------------- | ---------------------------------------- |
| `npm run build`         | Build dist/ from src/ and src-sdk/       |
| `npm test`              | Run the Jest test suite                  |
| `npm run test:coverage` | Run tests with coverage report           |
| `npm run lint`          | Check code with ESLint                   |
| `npm run format`        | Format code with Prettier                |
| `npm run format:check`  | Check formatting without modifying files |
| `npm run spellcheck`    | Spellcheck markdown files                |
| `npm run sdk --sdk=...` | Rebuild AWS SDK with custom services     |

## Project structure

```
src/            Source files deployed to Google Apps Script
src-sdk/        AWS SDK patches and build templates
dist/           Built output (generated by npm run build)
test/           Jest unit tests
build.js        Node.js build script
aws-sdk-js/     Git submodule for custom AWS SDK builds
```

## Making changes

### Adding a new service wrapper

1. Create `src/ServiceName.js` with wrapper functions
2. Follow the existing pattern: JSDoc, error handling with `return false`, promise-based
3. Create `test/ServiceName.test.js` with unit tests using the VM sandbox pattern
4. Add example functions to `src/Examples.js`
5. Update the services table in `README.md`
6. Update the default SDK list in `.npmrc` if the service needs to be included

### Modifying polyfills

Polyfills in `src/Polyfill.js` bridge the gap between browser/Node.js APIs and Google Apps Script's V8 runtime. When adding or modifying polyfills:

- Test in an actual Google Apps Script environment (the Jest tests mock GAS globals)
- Ensure the polyfill doesn't break the browser-compatible SDK build
- Document any limitations (e.g., the crypto polyfill is not cryptographically secure)

### Modifying SDK patches

The files in `src-sdk/*.part.js` are injected into the AWS SDK during the build. Changes here affect the core SDK behavior:

- `xhrClient.part.js` — HTTP client using `UrlFetchApp`
- `httpClient.part.js` — Routes between browser and GAS HTTP clients
- `xmlParser.part.js` — XML parsing using `XmlService`

Test patches carefully, as regex-based injection is sensitive to SDK version changes.

## Code style

- **ESLint** and **Prettier** are enforced. Run `npm run lint` and `npm run format` before committing.
- Use `const`/`let` (not `var`)
- Add JSDoc with `@param`, `@returns`, and `@example` to all public functions
- Use `Object.prototype.hasOwnProperty.call()` instead of `obj.hasOwnProperty()`
- Error handling: catch errors, log with `Logger.log(err, err.stack)`, return `false`

## Testing

All changes should include tests. The test suite uses Jest with a VM sandbox pattern to load Google Apps Script source files in a Node.js environment:

```js
const fs = require('fs');
const vm = require('vm');

function loadModule() {
  const code = fs.readFileSync(`${__dirname}/../src/YourModule.js`, 'utf8');
  const sandbox = {
    ...global,
    AWS: {
      /* mock AWS services */
    },
    Logger: { log: jest.fn() },
  };
  vm.createContext(sandbox);
  vm.runInContext(code, sandbox);
  return sandbox;
}
```

Run the full suite with `npm test`. Aim for test coverage on all public functions and error paths.

## Submitting changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-change`)
3. Make your changes and add tests
4. Run `npm test && npm run lint && npm run format:check`
5. Commit with a clear message describing the change
6. Open a pull request against `master`

## Reporting issues

Use [GitHub Issues](https://github.com/dxdc/aws-sdk-google-apps/issues) to report bugs or request features. Include:

- Steps to reproduce (for bugs)
- Expected vs actual behavior
- Google Apps Script runtime version (V8)
- Relevant error messages or logs
